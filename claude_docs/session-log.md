### 2026-03-02 (session 14 — Backend Stability Fixed, Dev Environment Hardened)
**Worked on:** Fixed persistent backend crash loop (`nodemon: not found` error). Root cause: `docker-compose.yml` was calling `npx nodemon packages/backend/src/index.ts` from workspace root `/app`, but pnpm doesn't hoist binaries to the root — npx couldn't find nodemon in PATH. Solution: Changed compose backend service command to `pnpm --filter backend run dev`, which respects pnpm workspace scoping and correctly resolves nodemon from `/app/node_modules/.bin` after install. Also moved `nodemon` and `tsx` from devDependencies to dependencies in `packages/backend/package.json` so they're present in the Docker image for containerized dev. Updated dev-environment skill with correct diagnosis and rebuild guidance (`docker compose down && docker compose up --build --no-cache`).
**Decisions:** nodemon and tsx are runtime deps for Docker container (live reload requirement) — kept in dependencies not devDependencies. pnpm --filter is the correct pattern for scoped commands in monorepo containers. --no-cache rebuild ensures fresh node_modules install.
**Next up:** Resume feature work — backend is stable. Real-user beta launch or Phase 9 (Creator Growth Platform) feature development.
**Blockers:** None.
